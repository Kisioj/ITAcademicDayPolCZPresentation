Daedalus - język skrytpowy Gothic I oraz Gothic II
Mówimy że Daedalus jest skryptowym językiem dzięki któremu można zaprogramować gameplay gry. Możemy nim zaprogramować zachowanie NPCów( rutyny ), zachowanie gdy zagadamy NPCa. Mówimy że dzięki skryptom w D komuniukujemy się z silnkiem gry, w D istnieją internale do silnika, np AI_output które sterują zachowaniem silnika. Typowa architektura jak dla gier koło 2000 roku, Quake 3 podobnie działa ( C QUAKE )
Przykład 

func void DIA_Bolec_IWantToGo_Drink ()
{

	if (Npc_HasItems (other, ItFo_Booze) >= 1)
	{
		B_GiveInvItems (other, self, ItFo_Booze, 1);
		B_UseItem (self, ItFo_Booze);
	}
	else if	(Npc_HasItems (other, ItFo_Beer) >= 1)
	{
		B_GiveInvItems (other, self, ItFo_Beer, 1);
		B_UseItem (self, ItFo_Beer);
	};
};
Mówimy że Daedalus jest podobny do C. Posiada funkcje, IFy, zmienne lokalne, zmienne globalne, możemy tworzyć ala klasy które są ograniczone, możemy tworzyć prototypy oraz instancje które działają podobnie do konstruktorów.
Ograniczenia - czyli dlaczego zdecydowaliśmy się napisać kompilator ?
Przewijamy dalej
1.
Kompilator jest wbudowany do pliku wykonywalnego gry ( Gothic2.exe )
Skrypter nie może szybko przekompilować rzeczy bez odpalania gry, musi przewinąć intro Nie możemy przez to łatwo postawić stagingu który by testował automatycznie czy skrypty są kompilowanlne w naszym pipeline
2.
Daedalus jest ubogi

Integracja z narzędziem do budowania moda
Kompilator będzie dopełnieniem narzędzia do budowania gry ( robocza nazwa gothic mod composer ). Dzięki temu nawet niedoświadczony skrypter będzie mógł w prosty sposób zacząć zabawę z modowaniem gothica
4.
